# Concert Tracker

## Description of the Project

This application is designed for company staff to manage and track concerts and their artists, venues, and promoters. It allows staff to 
create, read, update, and delete all 4 of those things. The application also allows staff to filter concerts by year, artist, venue, city,
and price aswell as generate reports to see revenue per venue, busiest venue and artist, average ticker price by year, and capacity. It aims
to solve the problem of tracking large amounts of concert data while making sure all of it is accurate and makes sense.

## User Stories

List the user stories that guided the development of your application. Format these stories as: "As a [type of user], I want [some goal] so that [some reason]."

- As a user, I want to be able to view all concerts, so that I can see every show currently stored in the system.
- As a user, I want to be able to add a concert, so that I can track a new show in the system.
- As a user, I want to be able to view a concert by its ID, so that I can see all information about a specific show.
- As a user, I want to be able to update a concert's ticket price, so that I can keep pricing information accurate.
- As a user, I want to be able to update the number of tickets sold, so that I can track ticket sales accurately.
- As a user, I want to be able to delete a concert, so that I can remove canceled or incorrect shows.
- As a user, I want to be able to manage artists, so that I can keep artist information up to date.
- As a user, I want to be able to manage venues, so that I can maintain accurate venue information.
- As a user, I want to be able to manage promoters, so that I can track the organizations responsible for concerts.
- As a user, I want to be able to search for concerts using different criteria, so that I can quickly find specific shows.
- As a user, I want to be able to search concerts using multiple conditions, so that I can narrow down my results.
- As a user, I want to be able to view revenue generated by each venue, so that I can identify the most profitable venues.
- As a user, I want to be able to identify the busiest venue and artist, so that I can see which ones host the most concerts.
- As a user, I want to be able to view average ticket prices by year, so that I can analyze pricing trends over time.
- As a user, I want to be able to view concert capacity utilization, so that I can see how full each concert is.
